# Task Brief: Task-001

## 目标
确定脚本命名规范并创建 1 个示例脚本，锁定输出格式。

## 相关 Playbook
- [practice-001] 必须遵守: 每个脚本的文件名与输出编号保持一致，便于检查。
- [anti-001] 必须避免: 在单个脚本中输出多个编号，违背独立脚本要求。
- [tech-001] 建议: 使用 f-string 或字符串拼接确保输出精确匹配 "hello world N"。

## 实现步骤
1. 阅读路线图确认 Task-001 目标，确定脚本命名规范并记录决定。
2. 按命名规范创建示例脚本，输出精确的 `hello world 1`（数字无前导零，无额外内容）。
3. 运行示例脚本验证输出与编号一致，记录结果以指导后续批量生成。

## 验收标准
- [ ] 已明确并记录的脚本命名规范可唯一对应编号。
- [ ] 存在一个示例脚本，运行输出正好为对应编号的 "hello world N"。
- [ ] 示例脚本执行验证通过，无额外输出或前导零。
