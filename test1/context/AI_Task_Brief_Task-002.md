# Task Brief: Task-002

## 目标
按规范批量生成 20 个脚本并验证输出与编号一致

## 相关 Playbook
- [practice-001] 必须遵守: 脚本文件名与输出编号保持一致，主入口仅输出对应编号一行
- [practice-002] 必须遵守: 在验证阶段检查文件名编号与输出编号一致
- [anti-001] 必须避免: 在单个脚本中输出多个编号

## 实现步骤
1. 以 `hello_world_1.py` 为模板，生成 `hello_world_2.py` 至 `hello_world_20.py`，确保文件名与编号一致
2. 在每个脚本中仅输出一行 `hello world <N>`，使用简单字符串或 f-string 保证精确匹配编号
3. 批量运行 20 个脚本，对比每个脚本输出与文件名编号一致，修正任何偏差

## 验收标准
- [ ] 存在 `hello_world_1.py` 至 `hello_world_20.py` 共 20 个脚本，命名符合 `hello_world_<N>.py`
- [ ] 每个脚本仅有一行输出 `hello world <N>`，且编号与文件名一致无多余输出
- [ ] 批量执行验证通过，未发现编号与输出不一致的脚本
